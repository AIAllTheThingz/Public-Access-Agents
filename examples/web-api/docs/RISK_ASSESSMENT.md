# Web API Composition Example Risk Assessment

## Purpose

This document demonstrates a project-level risk assessment for the fictitious example.

## Classification

- **Risk:** `moderate`
- **Rationale:** meaningful service, integration, security, or operational impact
- **Reversibility:** illustrative; no production state exists
- **Blast radius:** limited to documentation in this repository
- **Accountable approver:** not assigned

## Assets and concerns

- source and configuration integrity
- selected standards and manifests
- public or integration contracts where applicable
- fictitious data and evidence
- build and artifact integrity
- operational behavior described by the example

## Trust boundaries

- Internet or network clients
- Authentication provider
- Database or external service
- Container runtime and deployment environment

## Primary risks

- copying fictitious assumptions into a real project
- omitting applicable standards
- treating example evidence as actual evidence
- inconsistent root and nested instructions
- unsafe defaults at external boundaries
- secrets or sensitive values entering examples
- claiming validation or production readiness without proof

## Required controls

- reviewed standards selection
- explicit tailoring and limitations
- input and boundary validation
- least privilege
- safe error handling
- deterministic tests
- dependency and artifact controls
- repository validation
- accountable human review

## Evidence required before completion

- manifest and narrative consistency
- validated links and identifiers
- parsed JSON evidence
- review of unresolved placeholders and production values
- exact checks run
- checks not run
- known limitations and residual risks

## Residual risk

The principal residual risk is misuse: a reader can copy the example without reassessing actual architecture, data, risk, environment, or obligations. Documentation cannot prevent determined negligence, a recurring human feature.

## Reassessment triggers

- architecture change
- real deployment
- real data
- external users or systems
- identity or authorization
- persistence or messaging
- availability or recovery objectives
- regulatory or contractual obligations
