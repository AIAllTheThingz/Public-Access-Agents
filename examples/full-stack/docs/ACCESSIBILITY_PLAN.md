# Full-Stack Web Application Composition Example Accessibility Plan

## Purpose

This plan demonstrates accessibility expectations for the fictitious full-stack user interface.

## Target

Target WCAG 2.2 AA unless a stricter obligation applies.

## Required coverage

- semantic structure
- keyboard operation
- visible focus
- accessible names and labels
- form instructions and errors
- contrast and non-color cues
- zoom and reflow
- reduced motion
- screen-reader behavior
- status messages
- responsive layouts
- captions or alternatives for media when present

## Validation

Use both automated and manual methods. Automated tools do not prove conformance.

Manual checks should include keyboard-only navigation, focus order, focus visibility, screen-reader smoke testing, zoom/reflow, contrast, motion preferences, forms, errors, and dynamic updates.

## Evidence

Record pages and components tested, tools, browsers, assistive technologies, results, defects, exceptions, owners, and retest status.

## Limitations

No user interface exists in this example, so no accessibility test has been performed.
