# Worker Service Composition Example Data Handling

## Purpose

This document demonstrates data-handling questions for the fictitious example.

## Data summary

The fictitious worker processes job identifiers and synthetic payloads. Real customer, patient, employee, or credential data is prohibited.

## Classification

All included example values are synthetic and public within this repository. No real personal, confidential, regulated, authentication, or production operational data is allowed.

## Required decisions for real adoption

- data owner
- purpose and lawful or contractual basis where applicable
- classification
- fields and schemas
- collection source
- recipients and integrations
- storage locations
- access control
- encryption
- logging and analytics boundaries
- retention
- deletion and correction
- backup and recovery
- test-data strategy
- incident and breach handling

## Minimization

Collect and retain only what the actual use case requires. Do not copy production data into development, examples, tests, logs, screenshots, or support artifacts.

## Telemetry

Use identifiers and metadata only when necessary. Redact or omit payloads and secrets.

## Limitations

No real data inventory, privacy review, retention schedule, or access-control verification exists in this example.
