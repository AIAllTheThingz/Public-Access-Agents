# Cisco Networking Package Manifest

## Files

- `AGENTS.md` — mandatory product behavior and evidence rules
- `README.md` — scope, authority, adoption, lifecycle, validation, recovery, and limitations
- `MANIFEST.md` — package contents and acceptance
- `standards/OPERATIONS_AND_AUTOMATION_STANDARD.md` — implementation and operations controls
- `templates/ADOPTION_CHECKLIST.md` — adoption readiness
- `templates/REVIEW_CHECKLIST.md` — change and implementation review
- `templates/EVIDENCE_RECORD_TEMPLATE.md` — redacted evidence record
- `examples/ADOPTION_EXAMPLE.md` — fictitious non-production composition example

## Acceptance

- IOS, IOS XE, NX-OS, and IOS XR boundaries are explicit.
- Controller, intent, repository, template, and local configuration ownership is resolved.
- StackWise, VSS, vPC, chassis, management, lifecycle, security, upgrade, rollback, and actual-state controls are testable.
- Official sources are revalidated for the exact platform and date.
- Examples contain no credentials, real addresses, or production data.
- Repository validation passes.
