# Governance Standards

Governance standards apply across languages, platforms, frameworks, and project types.

Precedence:

1. Legal, regulatory, contractual, and safety obligations
2. Root repository instructions
3. Governance standards
4. Project profile
5. Language, discipline, platform, and framework standards
6. Scoped nested instructions

A lower-level standard may be stricter. It may not silently weaken a higher-level rule.
