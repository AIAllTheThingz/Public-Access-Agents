# Low-Risk Change Example

## Fictitious change

Correct wording in a non-normative README without changing behavior, links, IDs, or instructions.

## Classification

- Risk: low
- Reversible: yes
- Production impact: none
- Sensitive data: none
- Privilege: none

## Required governance

- scope review
- final diff review
- link validation if links changed
- honest completion statement

## Evidence

- changed file
- diff
- validation result
- statement that no normative rule changed

## Escalation triggers

Reclassify if the wording changes a requirement, precedence, approval, exception, or production gate.
