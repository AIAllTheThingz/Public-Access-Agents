# High-Risk Change Example

## Fictitious change

Migrate privileged identity data to a new service and database schema.

## Classification factors

- sensitive data
- privileged operations
- data migration
- hard-to-reverse behavior
- multiple systems
- production availability

## Required governance

- high-risk classification
- threat model
- architecture and data review
- security and privacy specialist review
- migration rehearsal
- backup and recovery evidence
- staged rollout and stop criteria
- independent review
- explicit production approval
- residual-risk acceptance

## Completion boundary

Successful unit tests do not establish migration, recovery, privacy, or production readiness.
