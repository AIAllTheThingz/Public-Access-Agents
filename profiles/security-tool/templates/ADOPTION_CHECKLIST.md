# Security Tool Project Profile Adoption Checklist

- [ ] Confirm the profile applies.
- [ ] Record whether it is primary or secondary.
- [ ] Classify risk.
- [ ] Identify users, operators, interfaces, data, identities, and environments.
- [ ] Select applicable language, discipline, framework, platform, virtualization, operating-system, and networking packages.
- [ ] Review all required disciplines.
- [ ] Promote conditional disciplines when their concern exists.
- [ ] Record omitted packages and justification.
- [ ] Define project decisions.
- [ ] Define architecture and trust boundaries.
- [ ] Define validation commands and evidence.
- [ ] Add nested instructions for distinct scopes.
- [ ] Assign owners, reviewers, approvers, and operators.
- [ ] Record exceptions and limitations.
- [ ] Validate links, IDs, manifests, and project behavior.
