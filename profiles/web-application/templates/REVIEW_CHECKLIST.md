# Web Application Project Profile Review Checklist

- [ ] Profile selection is justified.
- [ ] Canonical profile and package agree.
- [ ] Risk classification reflects actual impact.
- [ ] Required packages are included.
- [ ] Conditional packages were evaluated.
- [ ] Architecture and trust boundaries are accurate.
- [ ] Security and privacy controls are proportionate.
- [ ] Tests include negative and failure paths.
- [ ] Operations and release ownership are explicit.
- [ ] Nested instructions do not weaken shared controls.
- [ ] Evidence belongs to the reviewed artifact and environment.
- [ ] Checks not run are visible.
- [ ] Limitations and residual risks have owners.
- [ ] Production readiness is a separate decision.
