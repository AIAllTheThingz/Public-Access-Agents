# FastAPI Adoption Checklist

- [ ] The project profile is selected.
- [ ] The [Python language package](../../../languages/python/) is adopted.
- [ ] Relevant disciplines and platforms are selected.
- [ ] Exact framework, language, runtime, and build versions are declared.
- [ ] Application architecture and framework extension points are documented.
- [ ] Configuration sources, precedence, validation, and secrets are defined.
- [ ] Authentication and authorization behavior is defined.
- [ ] Lifecycle, async, concurrency, cleanup, and shutdown behavior is defined.
- [ ] Data access, transactions, and external integrations are defined where applicable.
- [ ] Error, health, telemetry, and diagnostic boundaries are defined.
- [ ] Test layers and validation commands are executable.
- [ ] Compatibility and migration commitments are recorded.
- [ ] Exceptions and inapplicable requirements are justified.
- [ ] Evidence location and reviewers are identified.
- [ ] No production secrets or internal values are present in the standards package.
