# Framework Package Collection Manifest

## Complete packages

- `frameworks/aspnet-core`
- `frameworks/react`
- `frameworks/angular`
- `frameworks/spring-boot`
- `frameworks/fastapi`

## Required collection files

- `frameworks/AGENTS.md`
- `frameworks/SKILL.md`
- `frameworks/README.md`
- `frameworks/MANIFEST.md`

## Package acceptance checks

Every framework directory must contain:

- `AGENTS.md`
- `README.md`
- `MANIFEST.md`
- framework-specific supporting standards
- `templates/ADOPTION_CHECKLIST.md`
- `templates/REVIEW_CHECKLIST.md`
- `templates/EVIDENCE_RECORD_TEMPLATE.md`
- `examples/ADOPTION_EXAMPLE.md`

Each package must preserve unique IDs, declare its underlying language, explain companion disciplines, provide useful adoption and validation guidance, and avoid production-specific values.
