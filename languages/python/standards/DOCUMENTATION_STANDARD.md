# Documentation Standard

## Requirements

- Document public APIs, configuration, prerequisites, examples, failure modes, and operational behavior.
- Explain why non-obvious decisions exist, not merely what syntax does.
- Keep commands executable and placeholders obvious.
- Update documentation in the same change as behavior.
- Do not claim validation that was not performed.

## Evidence

Completion evidence must identify the files and checks that demonstrate these requirements. Exceptions must follow the repository exception process and must not be implied by omission.
