# Security Standard

## Requirements

- Treat external input, configuration, files, network responses, and dependencies as untrusted.
- Use least privilege and secure defaults.
- Never commit secrets or expose them in logs, errors, examples, or test fixtures.
- Prevent injection by using structured APIs and parameterization.
- Validate authentication, authorization, cryptography, and transport assumptions at trust boundaries.

## Evidence

Completion evidence must identify the files and checks that demonstrate these requirements. Exceptions must follow the repository exception process and must not be implied by omission.
