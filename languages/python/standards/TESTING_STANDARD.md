# Testing Standard

## Requirements

- Add tests at the lowest useful level and include integration tests where boundaries matter.
- Test success, failure, boundary, authorization, cancellation, and idempotency behavior as applicable.
- Do not replace meaningful assertions with snapshot or existence-only checks.
- Keep tests deterministic and isolated from production systems.
- Report skipped or unavailable test coverage honestly.

## Evidence

Completion evidence must identify the files and checks that demonstrate these requirements. Exceptions must follow the repository exception process and must not be implied by omission.
