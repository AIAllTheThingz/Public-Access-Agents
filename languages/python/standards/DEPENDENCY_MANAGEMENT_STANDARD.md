# Dependency Management Standard

## Requirements

- Add dependencies only when their value exceeds maintenance and supply-chain cost.
- Prefer actively maintained, well-scoped dependencies from authoritative sources.
- Pin or constrain versions according to ecosystem norms and commit lock data where applicable.
- Review licenses, provenance, known vulnerabilities, install hooks, and transitive impact.
- Remove unused dependencies and document exceptional risk.

## Evidence

Completion evidence must identify the files and checks that demonstrate these requirements. Exceptions must follow the repository exception process and must not be implied by omission.
