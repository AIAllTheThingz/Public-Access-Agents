# Documentation Standard

- Document public items, safety invariants, feature flags, supported targets, configuration, examples, and failure behavior.
- Include runnable documentation tests where useful.
- Explain ownership, lifetime, FFI, and concurrency constraints that maintainers must preserve.
- Update documentation with behavior and API changes.
- Never claim checks that were not run.
