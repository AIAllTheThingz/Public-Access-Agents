# Security Standard

- Parameterize values and use allowlists for dynamic identifiers.
- Apply least privilege to users, roles, schemas, procedures, and service accounts.
- Do not embed credentials or sensitive production data.
- Protect row, tenant, and object authorization at the correct trust boundary.
- Review dynamic SQL, elevated execution contexts, ownership chaining, external access, and data export as high risk.
- Audit privilege and sensitive-data changes.
