# Observability Standard

- Capture migration identity, duration, affected objects, affected rows, and outcome without exposing sensitive data.
- Monitor blocking, deadlocks, query latency, errors, storage, replication, and resource saturation where applicable.
- Use stable query or migration identifiers for troubleshooting.
- Preserve useful database error context while redacting values.
- Document expected signals and recovery thresholds.
