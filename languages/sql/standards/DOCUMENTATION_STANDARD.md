# Documentation Standard

- Document engine, version, compatibility level, prerequisites, object purpose, parameters, result shape, side effects, permissions, transaction behavior, and failure modes.
- Explain migration order, data backfill, validation, rollback or forward recovery, and operational impact.
- Keep examples fictitious and free of sensitive data.
- Update application and operational documentation with contract changes.
- Never claim tested engines, data volumes, or plans that were not validated.
