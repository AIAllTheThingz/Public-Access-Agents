# Documentation Standard

- Document purpose, synopsis, parameters, environment variables, required commands, privileges, examples, exit codes, files changed, and failure behavior.
- Explain dry-run, confirmation, idempotency, rollback, and cleanup behavior.
- Keep examples safe and use fictitious targets.
- Update documentation with behavior changes.
- Never claim tests or platform support that were not verified.
