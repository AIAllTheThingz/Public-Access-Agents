# Documentation Standard

- Document engine and versions, prerequisites, authentication, backend, state ownership, module purpose, inputs, outputs, examples, deployment process, and recovery.
- Explain destructive behavior, replacements, import, moved resources, and migration steps.
- Keep examples fictitious and free of secrets.
- Update diagrams, runbooks, and upgrade notes with behavior changes.
- Never claim plan or apply validation that was not performed.
