# Java Adoption Checklist

- [ ] Exact JDK and build-tool versions are declared.
- [ ] Maven or Gradle wrapper and dependency strategy are declared.
- [ ] Supported platforms and compatibility boundaries are documented.
- [ ] Security-sensitive inputs and trust boundaries are identified.
- [ ] Formatting, static analysis, test, packaging, and runtime checks are configured.
- [ ] Secrets and production identifiers are absent from committed examples.
- [ ] Completion evidence requirements are integrated into the project workflow.
- [ ] An accountable maintainer reviewed the tailored standard.
