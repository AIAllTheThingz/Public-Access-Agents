# Testing Standard

## Requirements

- Add tests at the lowest useful level and integration tests where boundaries matter.
- Test success, failure, boundary, authorization, interruption, cancellation, and idempotency behavior as applicable.
- Keep tests deterministic and isolated from production systems.
- Test the packaged artifact and runtime configuration where deployment behavior matters.
- Report skipped or unavailable test coverage honestly.
