# Template Completion Criteria

## Collection completion

The collection is complete when:

- every catalog entry has a package
- every package has a stable template
- stable legacy paths remain present
- selection and customization guidance exist
- placeholder conventions are enforced
- lifecycle and compatibility are documented
- validation runs in permanent CI

## Package completion

A package is complete when:

- template scope is explicit
- use and non-use cases are documented
- every placeholder is documented
- template contains meaningful instruction
- review checklist matches the template
- example is completed and fictitious
- completion criteria are explicit
- related governance and schemas are linked
- common failure modes are documented
- compatibility is addressed
- validation passes

## Adopted record completion

An adopted record is complete when:

- no placeholders remain
- facts are authoritative
- scope and ownership are explicit
- decisions use defined vocabulary
- evidence is linked
- limitations are visible
- review and approval are traceable
- lifecycle and revisit triggers are defined
- applicable schema validation passes
- sensitive values are handled correctly

## Non-equivalence

The following are not equivalent:

- file exists = record complete
- headings filled = decision sound
- checklist checked = evidence genuine
- reviewer named = review performed
- approver named = approval authorized
- deployment passed = production ready
- schema passed = record truthful

This distinction is the difference between a template library and a stationery department.
