# Ubuntu Adoption Checklist

- [ ] Stable system, owner, Server/Desktop/flavor, release, kernel and architecture recorded
- [ ] Lifecycle, standard/Pro/ESM coverage, source-review date and upgrade owner recorded
- [ ] APT suites/components, signing trust, pinning, holds, phasing, PPAs and Snaps inventoried
- [ ] cloud-init, image, Landscape/configuration and local authority mapped
- [ ] AppArmor, firewall, identity/SSH, Netplan, storage/encryption and alternate access defined
- [ ] Desktop user data, accessibility and session impact addressed where applicable
- [ ] Transaction preview, canaries, batches, reboots, stop conditions and recovery defined
- [ ] Source/signature, dpkg lock, low-space, cloud-init, AppArmor, access, Netplan, boot and desktop tests exist
- [ ] Actual package/kernel/security/service/user/workload/backup/monitoring verification defined
- [ ] Checks not run and residual risk will be reported
