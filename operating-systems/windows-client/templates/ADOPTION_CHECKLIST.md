# Windows Client Adoption Checklist

- [ ] Stable device identity, custodian class, hardware, edition, release, build and architecture inventoried
- [ ] Join, enrollment, MDM, Configuration Manager, Group Policy, security and update authorities mapped
- [ ] Windows 10 LTSC/ESU/exception or Windows 11 servicing status verified
- [ ] Hardware, driver, application, language, accessibility and security-tool compatibility tested
- [ ] BitLocker protection and authorized recovery-key escrow verified
- [ ] Secure Boot, TPM, Defender, firewall, baseline and LAPS states defined
- [ ] User data, privacy, legal hold, backup/sync and recovery handled
- [ ] Deployment rings, exclusions, deadlines, notifications, restarts and stop conditions defined
- [ ] Offline, stale, denied, safeguard, low-space, partial, restart and recovery tests exist
- [ ] Reset, wipe, retire, reimage and account-removal authority separated
- [ ] Device-side actual-state and user/workload validation defined
- [ ] Redacted evidence, checks not run and residual risk will be retained
